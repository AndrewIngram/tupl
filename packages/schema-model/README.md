# `@tupl/schema-model`

Logical schema DSL, normalization, and provider binding validation for `tupl`.

This package backs `@tupl/schema`; most application code should prefer `@tupl/schema`.

Use this package directly when you need lower-level schema authoring, normalization, enum-link resolution, or provider-binding validation.
It is not the normal adapter-authoring entrypoint; third-party providers should usually use `@tupl/provider-kit` instead.
